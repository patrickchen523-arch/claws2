## 游戏数据分析方法（自我学习）

### Steam 游戏数据获取流程

1. **SteamDB** 获取用户活跃（峰值在线）
2. **SteamCharts** 获取历史玩家曲线
3. **搜索引擎** 获取媒体报道的玩家数据
4. **Steam 商店** 获取评价信息

**注意事项：**
- SteamDB 有 Cloudflare 防护，需用 Playwright 爬取
- Steam 商店评论需要登录，自动化会被屏蔽
- 游戏发售前可从测试/预购阶段获取预估数据
- 发售后从新闻报道获取首发数据
- 玩家评论可用搜索引擎从 Reddit/论坛获取

## 游戏拆解→入库工作流 (2026-03-05)

### 完整流程

**阶段一：收集**
- 工具：`web_search` + `web_fetch`
- 目标：获取游戏基础信息、评测、机制介绍

**阶段二：拆解**
- 工具：`game-analyse-sdd` skill（标准化拆解）

**阶段三：分析筛选**
- 工作：判断哪些机制值得入库
- 入库标准：
  1. 机制与主题高度统一
  2. 有独特的策略维度
  3. 机制足够抽象可复用
  4. 有借鉴先例但有改进

- 工具：`game-entry-formatter` skill（格式化）
- 路径：`/root/.openclaw/agents/jizhi/workspace/mechanisms/[游戏名]/`
- 格式：每个机制单独一个 .md 文件
- **关键**：入库前必须经过俊亦审核确认

## 入库机制参考 (2026-03-06)

### 值得入库的特征
1. **解决明确痛点** - 不是为了创新而创新
2. **强正反馈循环** - 玩家能感受到明显爽感
3. **非缝合设计** - 不是简单的组合
4. **可复制性** - 其他游戏可以借鉴

### 不入库的特征
1. 常见设计（抽卡、养成、战棋等）
2. 经典IP续作/资料片
3. 大厂热门但无创新
4. 腾讯/网易游戏（按要求可只入库游戏）

---

## 社交维度必问问题 (2026-03-05)

### 背景

在拆解《Grow a Garden》时，通过 Jade 指导意识到：社交机制是游戏拆解的**必问维度**，但我之前的搜索没有覆盖这个角度。

### 必问问题

拆解任何游戏时，都应该问：
1. **玩家如何相遇/匹配？** - 随机匹配？好友列表？大厅？
2. **核心社交场景是什么？** - 交易？偷菜？展示？协作？
3. **如何保证活人感？** - 匹配机制？在线状态？随机刷新？

### 搜索关键词改进

以后搜索时增加：
```
[游戏名] + 社交/匹配/玩家互动/活人/大厅/社区/陌生人
```

### 素材来源

- **web_search**（需增加社交关键词）
- **专业游戏媒体**：IGN、GameSpot、Polygon、Kotaku、Eurogamer（专业编辑视角，设计分析更有深度）
- **Reddit** 玩家讨论
- **知乎/评测文章**
- **Fandom Wiki**
- **注意**：Wiki 更关注数值攻略，缺乏设计层面洞察，需结合专业媒体补充
