# 2026-03-11 工作日志

## 新项目：代号 Pokotraction（宝可梦搜打撤）

### 游戏概念
- **类型**：单人回合制PVE搜打撤
- **核心**：宝可梦战斗系统 + 体力条限制
- **参考**：宝可梦战斗、三角洲的赚钱、梦幻合宠

### 核心循环
1. **战斗内**（秒级）：宝可梦选招回合制战斗、多精灵切怪
2. **单局**（分钟级）：体力值限制，环形地宫，越深战利品越丰富
3. **长线**：弱数值系统，宠易满数值但有方向分化和打书深度

### 经济循环
- 捉宠出售（对标 Pokemmo）
- 治疗精灵花销（对标搜打撤修装备）
- 合宠养宠（对标梦幻）
- 消耗道具

### Gemini 建议的 MVP 模块

**1. 核心战斗**
- 3种互相克制的属性（水、火、草）
- 预设3-4只精灵，固定技能
- 回合制逻辑、血条计算、属性克制、切怪功能

**2. 体力驱动的地宫探索**
- 3层环形节点地图（外层、中层、核心层）
- 移动和战斗回合都扣除体力
- 外层安全/低级怪 → 核心层精英怪
- 撤离点设定

**3. 单局结算**
- 背包容量限制
- 死亡/体力耗尽 = 战利品清零
- 战利品→金币→治疗消耗

### 待验证的核心体验
玩家在中层时，看着剩余体力，选择继续深入还是撤退的"患得患失"

### 讨论结论（确定版）

**1. 体力系统**
- 默认体力：100
- 移动消耗：1体力/格
- 战斗消耗：遇敌固定扣4点 + 每回合扣1点
- 体力0~-24：每消耗1体力额外扣除4%最大血量
- 体力≤-25：判定撤离失败
- 逃跑扣除：40体力

**2. 地图系统**
- 双层结构：外层Node Map + 内层8x8矩阵
- 明雷：任务点/BOSS
- 暗雷：草地/水路，概率遇怪
- 地形：空地、门、草地、水路、战利品点

**3. 背包系统**
- 3x4 = 12格，1格=1物品（保留size属性）

**4. 战斗系统**
- 传统回合制
- 切宠当回合不能攻击

**5. 难度梯度**
- 外层/中层/核心，数值递增

**6. 初始阵容**
- 3只基础宠

**7. 经济**
- 战利品：白色1块，每稀有度×5
- 治疗：早期固定5元/次

### 后续机制（MVP后开发）

**受伤残值系统**
- 目的：防止切治疗宠无限制回血
- 机制：
  - 受伤残值 = 本次对局可治疗的最大HP
  - 初始 = 进入地图时的HP
  - 每次受伤后，受伤残值减少本次伤害的20%
  - 例：满血100被打到1，受伤残值减少约20，最多只能回到80%

## 养成系统设计文档（待开发）

### 1. 核心情绪：技能"资产化"与装备恐惧

**局外仓库（Stash）**
- 玩家成功带出的技能书存入仓库
- 仓库技能书可重复使用

**战前打书（Loadout）**
- 每次下本前可给精灵装备技能
- 相当于搜打撤的"起半装/全装"

**死亡惩罚**
- 精灵不会死，但身上额外技能被洗掉
- 只保留自带基础技能（火花/水枪/藤鞭）

**玩家心理**
- 带好技能 → 怕死
- 穿好装备怕爆，穿破烂打不过
- 极致博弈驱动力

## PRD 模块化整理（正式版）

### 1. Game Loop（核心循环）
- **准备阶段**：局外界面 → 开局（扣金币治疗）→ 进入地图
- **探索阶段**：8x8 房间内移动 → 遇敌/拾取战利品/使用门传送
- **战斗阶段**：回合制战斗 → 胜负结算
- **结算阶段**：撤离成功（战利品入包）→ 失败（战利品清零）→ 回到局外

### 2. Resource Management（资源管理）
- **体力系统**：
  - 初始 100 点
  - 移动：-1/格
  - 遇敌：-4点
  - 战斗每回合：-1点
  - 逃跑：-40点
  - 0~-24：每扣1体力，全队扣4%最大HP
  - ≤-25：判定失败
- **经济**：
  - 战利品价值：白色50 → 绿色150 → 蓝色500 → 紫色1500 → 金色5000 → 红色20000
  - 治疗费用：20金币/次

### 3. Map & Exploration（地图系统）
- **宏观层**：Node Map，3-5节点单向路线
- **微观层**：8x8 矩阵
- **地形**：空地（安全）、草地/水路（暗雷）、岩石（障碍）、门（传送）、战利品点、明雷
- **撤离点**：每路线有1-2个撤离点

### 4. Combat System（战斗核心）
- **速度轴**：按 Speed 排序决定行动顺序
- **行动**：技能/换宠/道具/逃跑（各消耗当前回合）
- **切宠规则**：换宠当回合不能攻击
- **敌人**：本地预设镜像配置

### 5. Entities & Data Structure
- **Pet**: pet_id, name, element, max_hp, current_hp, speed, skills
- **Item**: item_id, name, type, width, height, value, description
- **技能**: skill_id, name, type, element, power, cooldown, effect
- **属性克制**: Water→Fire×1.5, Fire→Grass×1.5, Grass→Water×1.5
