# 城市天际线2 机制知识卡

**品类原型**：城市建造-like
**创新定性**：原型+结构创新
**分析日期**：2026-03-18

---

## 结构创新

### 真实产业链经济
- **来源子模块**：3.5 制作/合成系统 + ADD-2 供需物流
- **做法**：9类原材料（木材/谷物/牲畜/蔬菜/棉花/原油/矿石/煤炭/石材）→中间品（金属/纺织/化学/石化等）→消费品（食品/饮料/电子/车辆等）+办公区无形商品（软件→电信/金融/媒体）；本地消费优先，盈余出口；出口距离衰减利润
- **解决的问题**：城建品类"划区划→自动产税→收钱→扩张"的经济过于扁平，中后期无经济决策空间
- **协同关系**：与发展树（乘法——发展树限制产业链展开节奏，产业链收入驱动发展树推进）、与市民就业（链式——产业需要对应教育水平工人→教育培养→就业→产出→消费闭环）、与进出口贸易（乘法——产业链创造产品×贸易将盈余变现=正反馈循环）
- **移植注意**：产业链复杂度对新手信息量大，必须配套产业链可视化工具（CS2本身缺乏此工具是主要痛点）；链条断裂的连锁反应难以定位

### 发展树解锁系统
- **来源子模块**：4.3 技能树/天赋系统
- **做法**：11个服务分支（道路/电力/水/医疗/垃圾/教育/消防/警察/交通/公园/通讯）×4层深度（Tier1=1点→Tier4=8点），发展点从20级里程碑获取（每级1→30点），需策略性分配
- **解决的问题**：城建品类"人口达标→自动全解锁→所有玩家发展路径趋同"的策略深度不足
- **协同关系**：与里程碑XP（链式——建设行为→XP→里程碑→发展点→解锁→更多建设）、与地图瓦片购买（乘法——发展树决定"能建什么"×地图购买决定"在哪建"=双维度规划）
- **移植注意**：后期点数溢出导致科技树意义消失，需设计点数持续消耗机制或无限深度分支；早期错误投点可能导致关键服务缺失

### 模块化建筑升级
- **来源子模块**：4.4 装备强化
- **做法**：服务建筑可加装功能模块——消防站+车库(+2消防车)/+停机坪；学校+翼楼(+100-500学生)/+体育馆(+幸福感)/+图书馆(+毕业率)；污水厂+处理模块/+过滤系统；每个模块有独立效果和占地
- **解决的问题**：城建品类"服务不足只能重复建同类建筑→空间浪费+管理负担"的空间管理痛点
- **协同关系**：与市民需求（弥补——需求增长制造压力×模块化提供不拆不建的扩容方案缓解压力）、与覆盖层数据（链式——数据定位问题→针对性加模块→验证效果）
- **移植注意**：模块不可拆卸（决策不可逆），需配套明确的空间预留提示；模块选择需足够差异化（+容量 vs +范围 vs +功能buff）才有决策价值

## 设计精华

> 产业链经济 × 发展树 × 模块化建筑升级的三角关系——产业链提供"建什么"的经济深度，发展树提供"何时建"的时序策略，模块化升级提供"怎么管"的设施精细度。三者将城市建造从"画区划等成长"转变为"经济规划+发展策略+设施管理"的主动决策体验。

## 体验支柱

| 支柱 | 支撑机制 | 核心协同 |
|------|---------|---------|
| 经济规划深度 | 产业链经济 + 进出口贸易 + 市民就业 + 税收体系 | 产业链×贸易（乘法）=经济正反馈；产业链×就业（链式）=经济闭环 |
| 渐进式发展规划 | 发展树 + 里程碑XP + 地图瓦片购买 + 签名建筑 | 发展树×地图购买（乘法）=双维度规划决策；里程碑→签名建筑（链式）=连续目标驱动 |
| 精细化设施管理 | 模块化升级 + 市民需求 + 覆盖层数据 | 模块化×需求（弥补）=动态平衡；覆盖层→模块化（链式）=精准优化 |

## 承重墙机制

| 机制 | 拿掉后的影响 |
|------|------------|
| 产业链经济 | 支柱1崩塌（经济退化为划区划收税），支柱2部分弱化 |
| 发展树 | 支柱2崩塌（解锁退化为自动全解锁），支柱1部分弱化 |
| 模块化建筑升级 | 支柱3崩塌（设施管理退化为重复建造） |
| 里程碑XP | 支柱2崩塌（发展树失去点数来源） |
