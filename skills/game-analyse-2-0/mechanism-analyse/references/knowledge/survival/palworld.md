# 幻兽帕鲁 机制知识卡

**品类原型**：生存建造-like + 怪物收集混合型
**创新定性**：品类突破
**分析日期**：2026-03-19

---

## 结构创新

### 帕鲁劳工系统
- **来源子模块**：1.5 状态效果 + 3.5 制作系统 + 基地系统
- **做法**：被捕获帕鲁按12种工作适性(Lv1-4)自动分配到基地工位(伐木/挖矿/种植/运输/制作/生火/浇水/发电/冷却/制药/采集/放牧)；SAN值(心理健康)系统约束无限压榨(过劳→抑郁→拒绝工作)；病症系统(感冒/扭伤/溃疡/骨折/虚弱/抑郁)增加管理复杂度
- **解决的问题**：生存建造"手动采集太枯燥" + 怪物收集"抓了100只只用6只"的双重痛点
- **协同关系**：与帕鲁捕获（乘法——劳工让已有帕鲁有价值×捕获提供来源=每次探索都强化基地）、与SAN值（乘法——分工创造优化空间×SAN值防止无限压榨=管理策略深度）、与浓缩升星（乘法——Rank4所有工作+1=升星直接提升产能）
- **移植注意**：帕鲁AI寻路经常卡住是主要体验问题；SAN值恢复如果太容易→约束失效，太难→玩家觉得被惩罚

### 帕鲁一体多用设计
- **来源子模块**：跨系统
- **做法**：同一只帕鲁同时可担任战斗搭档(Partner Skill武装化=喷火器/冲锋枪/盾牌)、基地工人(12种适性)、坐骑(地/海/空)、食材(可屠宰)、牧场产出(被动资源)、交易商品
- **解决的问题**：怪物收集"怪物只有战斗一个用途→大量怪物沦为收集品"的资源浪费问题
- **协同关系**：与帕鲁劳工（乘法——一体多用让每只帕鲁有多维价值×劳工系统让"工作"成为高频使用场景=帕鲁价值最大化）、与牧场（链式——捕获→放牧→产出→制作→捕获更多）
- **移植注意**：功能过多导致单一功能深度不足（战斗不如Pokemon/基地不如异星工厂）；"万能"设计需要控制各功能间的平衡

## 设计精华

> 帕鲁劳工×一体多用×浓缩升星三角——劳工系统让帕鲁是员工，一体多用让帕鲁是万能工具，浓缩升星让重复捕获有目标。"帕鲁即一切"的设计让整个游戏世界围绕帕鲁运转。

## 体验支柱

| 支柱 | 支撑机制 | 核心协同 |
|------|---------|---------|
| 帕鲁改变一切 | 一体多用+劳工+Partner Skill+坐骑+牧场 | 一体多用×劳工（乘法）=帕鲁价值最大化 |
| 我的帕鲁工厂 | 劳工系统+工作适性+SAN值/病症+基地建设 | 分工×SAN值（乘法）=人力资源管理深度 |
| 再来一只 | 浓缩升星+图鉴+配种+被动技能遗传 | 升星×劳工（乘法）=Rank4工作+1直接提升产能 |

## 承重墙机制

| 机制 | 拿掉后的影响 |
|------|------------|
| 帕鲁劳工系统 | 支柱1、2崩塌，游戏退化为标准生存+战斗宠物 |
| 帕鲁一体多用 | 支柱1崩塌，帕鲁角色单一化 |
| SAN值/病症 | 支柱2崩塌，基地管理无深度 |
