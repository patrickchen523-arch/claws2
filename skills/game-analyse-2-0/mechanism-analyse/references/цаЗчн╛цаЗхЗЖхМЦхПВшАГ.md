# 机制库标签标准化参考文档

> 本文档用于指导机制卡的标签标准化工作
> 标签分为三个层级：玩法层级、机制设计层级、痛点层级

---

## 一、玩法层级标签（必填）

### 1. 对抗类型

| 标签 | 说明 | 示例 |
|-------|------|------|
| PVE | 玩家 vs 环境/AI | 副本、Boss战 |
| PVP | 玩家 vs 玩家 | 天梯、排位 |
| PvPvE | 先PVE后PVP | 搜打撤、逃离塔科夫 |
| GVG | 公会 vs 公会 | 公会战、阵营战 |
| 异步竞技 | 非实时对抗 | 异步PVP、离线收益比较 |

### 2. 玩法人数规模

| 标签 | 说明 | 示例 |
|-------|------|------|
| 单人 | 1人游戏 | 单机RPG、单人模式 |
| 双人 | 2人 | 本地双人、双人合作 |
| 小队 | 3-7人 | 三人组队、5v5、四人副本 |
| 多人 | 8人+ | 公会战、MMO战场、大逃杀 |

**填写规则**：一个机制可能同时适用多个规模（如"单人、小队"），用顿号分隔。

### 3. 内容消耗模式

| 标签 | 说明 | 示例 |
|-------|------|------|
| 一次性 | 线性流程，关卡制 | 单机剧情 |
| 周期性 | 赛季、轮换活动 | 赛季通行证、月度活动 |
| 可重复 | 无尽刷宝 | Roguelike、刷宝 |
| UGC | 用户生成内容 | 地图工坊、编辑器 |

### 4. 生命周期阶段

| 标签 | 说明 | 示例 |
|-------|------|------|
| 新手引导期 | 教程、前期成长 | 新手任务、引导关卡 |
| 成长中期 | 中期BD构建 | 天赋树、装备积累 |
| Endgame | 毕业、高难度 | 高难副本、排名竞争 |

### 5. 操作属性

| 标签 | 说明 | APM参考 |
|------|------|----------|
| 即时操作 | 实时操作 | APM>60 |
| 半即时操作 | 有操作窗口 | APM 15-60 |
| 回合操作 | 回合制思考 | APM<15 |
| 放置操作 | 挂机、单手操作 | APM<5 |

### 6. 时间属性

| 标签 | 说明 | 示例 |
|-------|------|------|
| 长时间 | >60分钟/局 | 硬核生存、深度策略 |
| 中规中矩 | 10-60分钟/局 | 标准MOBA、RPG副本 |
| 碎片化 | 3-10分钟/局 | 休闲手游、短对局 |
| 高度碎片化 | 1-3分钟/局 | 地铁对局、等待间隙 |

### 7. 玩法重度

| 标签 | 说明 | 示例 |
|-------|------|------|
| 核心玩法 | 主体玩法 | 战斗系统、策略深度 |
| 副玩法 | 辅助玩法 | 小游戏、抽卡系统 |
| 轻度玩法 | 休闲内容 | 叙事、对话、社交 |

### 8. 玩法独立性

| 标签 | 说明 | 示例 |
|-------|------|------|
| 高耦合 | 核心数值系统 | 经济、装备、BD |
| 低耦合 | 独立小游戏 | 钓鱼、换装、聊天 |

### ⚠️ 重要规则

**遇到未收录的品类、情绪、交互关系时**：
- 暂时使用"其他"占位
- 确认后更新模板，再修改机制卡

### 情绪反馈判定标准

**重要规则**：请阅读机制描述，模拟你是玩家。选择强度最高的1个核心情绪。

**注意事项**：
- **关联性校验**：
  - 如果选了"烧脑"，请检查16乐趣里是否有"策略"或"解谜"
  - 如果选了"治愈"，请检查16乐趣里是否没有"竞技"和"挑战"
- **互斥检查**："爽快感"与"烧脑"通常不同时作为核心反馈出现，请甄别主次

| 标签 | 判定题 | 心理/生理表现 |
|------|--------|---------------|
| **爽快感** | 玩家是否在短时间内进行了高频的操作输入，且伴随大量的视觉特效反馈？ | 肾上腺素飙升，感到"宣泄"和"停不下来" |
| **惊喜随机** | 结果是否超出了玩家的预期（由于系统概率），且该结果显著利好于玩家？ | 尖叫、截图分享、"今天运气真好" |
| **紧张感** | 玩家是否处于高风险环境下（如资源枯竭、血量见底、限时倒计时），且一步操作失误就会导致巨大损失？ | 屏住呼吸、手心出汗、心跳加速 |
| **烧脑** | 玩家是否陷入了长时间的沉默思考、反复对比不同方案，甚至需要动用纸笔或计算器？ | 眉头紧锁，在"我懂了"的那一刻爆发 |
| **恐怖/微恐** | 机制是否利用了未知感、幽闭环境、突发音效或视觉上的扭曲来让玩家感到"不安全"？ | 肌肉紧绷、下意识地想缩在屏幕远端 |
| **节奏沉浸** | 玩家是否陷入了一种重复但有节奏的行为（如音游、钓鱼、循环刷怪），导致忘记了时间的流逝？ | 感到"心流"，周围的环境噪音被自动过滤 |
| **治愈/放松** | 机制是否排除了"失败"和"死亡"的可能，且美术/音乐表现柔和，不给玩家任何压迫感？ | 肩膀放松、深呼吸、感到心情平静 |
| **成就感** | 玩家是否在完成了一个高难度目标（或长期积累任务）后，获得了一个象征荣誉的反馈？ | 如释重负、感到"所有的付出都值了" |

### 交互关系判定标准

**核心思路**：聚焦于"玩家行为对另一个人的影响结果"。

**甄别准则**：
- 如果同时存在合作与对抗（如：组队PVP），标签应标为"合作"、"对抗"
- 如果只是在主城能看见人，但机制是自己打怪，请务必甄别是否为"无交互"
- "欺骗"是高阶标签，除非机制卡里明确提到了"身份"、"伪装"、"说谎"，否则不要轻易打这个标签

| 标签 | 判定题 | 核心逻辑 |
|------|--------|----------|
| **合作** | 如果其中一名玩家表现极差，另一名玩家是否会直接导致任务失败或利益受损？ | 利益共同体。你的成功就是我的成功。 |
| **对抗** | 玩家之间是否可以互相造成伤害、位移、资源扣除，或者存在"你多得一个，我就少得一个"的零和博弈？ | 利益对立面。你的损失就是我的收益。 |
| **欺骗** | 机制是否涉及"隐藏身份"、"虚假信息传递"或"背叛机制"？ | 信息差博弈。如果玩家之间完全诚实，这个机制是否会立刻失效？ |
| **交易/互惠** | 玩家是否通过"交出某样东西"来换取"另一样东西"，且这个过程是自愿且双赢的？ | 等价交换。双方在互动后，资源总量不变或优化，但分配方式改变。 |
| **无交互** | 在该机制进行时，其他玩家的存在（无论是操作还是数值）是否对当前玩家的体验产生0影响？ | 平行时空。即使服务器里只有我一个人，这个机制的玩点也完全不变。 |

### 16乐趣标签（选2个最重要的）

| 标签 | 判定题 | 甄别点 |
|------|--------|--------|
| **解谜/破解** | 玩家是否需要通过观察环境、推导逻辑规律来获得一个"唯一解"或"特定过关方式"？ | 重点在于信息不对称的消除。一旦知道了答案，第二次玩是否就失去了90%的乐趣？ |
| **策略** | 玩家是否需要在多个"正确方案"中，根据资源、环境或对手行为，通过长线规划选出"最优解"？ | 重点在于动态博弈和资源分配。它通常不只有唯一解，且需要持续的决策输入。 |
| **考究** | 该机制是否提供了大量的文字、物品描述或隐藏细节，且这些内容不直接影响胜负，但会补完世界观？ | 玩家是否在玩"拼图"，通过碎片信息去还原真相。 |
| **操作** | 玩家的"反应速度"、"肌肉记忆"或"输入精准度"是否是决定胜负的核心因素？ | 哪怕玩家脑子里知道怎么赢，但如果手跟不上，是否就会失败？ |
| **挑战** | 机制是否刻意设置了极高的失败门槛，要求玩家必须经历"失败-学习-再尝试"的反复循环？ | 重点在于"克服逆境"。它是针对玩家意志力的磨炼，而非简单的杀时间。 |
| **竞技** | 玩家的成就感是否直接来源于"在规则下击败了另一个真实人类"或"在排行榜上压倒了他人"？ | 强调横向对比。如果没有其他玩家作为参照物，这个乐趣是否消失？ |
| **代入/剧情** | 玩家是否被要求扮演一个特定角色，且该角色的情感、对白或命运发展是吸引玩家玩下去的主因？ | 机制是否在服务于"讲故事"。如果把文字和美术全删掉只留数值，玩家是否还会觉得有趣？ |
| **创造** | 玩家是否利用游戏提供的原件，输出了一个"世界上独一无二"的成果（如关卡、建筑、逻辑回路）？ | 玩家的身份是否从"消费者"变成了"生产者"。 |
| **模拟现实** | 机制是否严格遵循现实世界的物理规律、职业逻辑或社会规则，以追求"真"而非"爽"？ | 如果现实中能做这件事，游戏中是否也采用了类似的步骤和限制？ |
| **养成/装扮** | 玩家是否对某个对象（角色、基地、宠物）投入了大量"情感或审美支出"，目的是让它看起来更好看或更符合个性？ | 重点在于"情感寄托"和"外在表现"。 |
| **成长变强** | 玩家是否通过消耗资源，获得了角色数值（攻击力、血量等）的永久提升？ | 重点在于"效率的提升"。以前打不过的怪，现在能一刀秒，这就是变强。 |
| **收集** | 机制是否提供了一个"清单"或"图鉴"，并鼓励玩家将其中缺失的空格填满？ | 重点在于"占有欲"。玩家是否为了凑齐一套不一定有实际用途的东西而努力。 |
| **盈利/赚钱** | 机制的核心目标是否是"低买高卖"、"资产增值"或"建立经济帝国"？ | 数字的增长本身就是目的，而不仅仅是变强的手段。 |
| **探索/高自由** | 玩家是否在没有明确指引的情况下，因为好奇心驱使去到了地图的死角，并获得了意外的反馈？ | 重点在于"未知"。玩家是否在走策划没有在小地图上标出来的路。 |
| **随机** | 结果是否由系统的随机数（RNG）决定，导致玩家在操作完全一致的情况下获得截然不同的结果？ | 重点在于"赌徒心理"。是否有"出货了"或"太非了"的情绪波动。 |
| **社交** | 玩家是否必须（或非常倾向于）通过与其他玩家交谈、组队或交换资源来达成目标？ | 重点在于"人与人的连接"。如果把所有玩家换成AI机器人，乐趣是否会大幅下降？ |

## 三、标签存储格式

### 机制卡存储格式

在机制卡中使用以下格式存储标签：

```markdown
## 标签信息

### 玩法层级
- 对抗类型：PVP
- 玩法人数规模：单局5人/单队5人
- 内容消耗模式：可重复
- 生命周期阶段：Endgame
- 操作属性：即时操作
- 时间属性：中规中矩（10-60分钟）
- 玩法重度：核心玩法
- 玩法独立性：高耦合

### 机制设计
- 原型品类：射击-搜打撤
- 16乐趣标签：策略、竞技、挑战
- 情绪反馈：紧张感、爽快感
- 交互关系：对抗
- 特色机制：撤离机制、装备带入带出

### 玩家痛点
- 挫败感痛点
- 经济成本痛点
```

## 四、判断规则

### 判断顺序

1. **先判断玩法层级** - 根据游戏核心循环判断
2. **再判断机制层级** - 根据机制所属品类判断
3. **最后判断痛点** - 根据机制解决什么问题判断

### 判断技巧

- 如果网上搜索无法确认标签 → 标注"无法判断"
- 如果机制跨越多个分类 → 选择最核心的1-2个
- 如果机制同时服务多个玩法 → 用"/"分隔
