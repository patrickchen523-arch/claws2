# mechanism-evaluate - 游戏创新评价

本 skill 基于 mechanism-analyse 的扫描报告，用品类原型/结构创新/包装层三层框架判断游戏的创新层次。

**核心价值**：不是从零分析游戏，而是站在 mechanism-analyse 的 57 模块扫描数据上做创新判断——每一个结论都有具体机制证据支撑。

---

## 一、使用方式

### 触发词
- "评价这份扫描报告"
- "单独评价 [游戏名] 的创新"
- "只做创新评价"
- "基于这份报告判断创新层次"

### 输入要求
- **默认输入**：对话上下文中的 mechanism-analyse 模式一扫描报告
- **指定文件**：用户可提供报告文件路径
- **必须是模式一（机制扫描）报告**，包含机制清单表格和综合叙述

---

## 二、执行流程

### Step 1: 读取 mechanism-analyse 报告

从对话上下文或用户指定路径读取模式一扫描报告。提取：

- **基础信息**：游戏名、品类、分析日期、信息来源
- **机制清单表格**：每个系统下的子模块表格（子模块 / 是否存在 / 做法概述 / 值得吸收 / 优先级 / 理由）
- **综合叙述**：机制特色、Top 5 值得吸收的机制、品类对比观察、风险提示

**完整度检查**：统计标记为"信息不足"的子模块占比。
- 低于 10%：置信度 = 高
- 10%-20%：置信度 = 中
- 超过 20%：置信度 = 低，在评价卡片中标注

---

### Step 2: 原型识别

读取 `references/genre-archetype-dictionary.md`，从报告中识别游戏的品类原型。

**核心循环证据提取**：重点看以下四大系统的 ✓/~ 项和做法概述：
- 战斗系统（1.1-1.8）：战斗核心是什么？操作型还是策略型？
- 任务系统（2.1-2.6）：驱动玩家的是什么？剧情？日常？探索？
- 养成系统（4.1-4.7）：成长结构是什么？等级？装备？角色收集？
- 关卡系统（6.1-6.6）：游戏空间是什么？开放世界？线性关卡？随机生成？

**匹配流程**：
1. 先查品类原型词典，找最接近的原型
2. 如果能匹配，直接引用原型名称和标志特征
3. 如果两个原型同等匹配，写成"A+B混合型"
4. 如果词典中没有匹配项，说明可能是新原型或极小众品类

**产出句式**：
> 这款游戏的品类原型本质上是一个 ______ 型产品，底层驱动力来自 ______。

**防误导**：
- 不要被题材误导（三国皮 ≠ SLG）
- 不要被 UI 误导（横版 ≠ 动作游戏）
- 不要把某个单点机制误判为原型变化（加了个钓鱼 ≠ 模拟经营）
- 不要只因为"多了一个系统"就说它是新品类

---

### Step 3: 补充搜索（最多 3 次）

仅在以下情况触发补充搜索：
- 原型识别不确定（两个原型都像）
- 报告中多个子模块做法异常（偏离品类常规）
- 需要确认某个做法是否为品类首创

**搜索关键词模板**：
1. `[游戏名] + [最接近原型的代表产品] + 对比/区别` — 确认原型差异
2. `[游戏名] + [异常机制名] + 创新/首创` — 确认结构创新还是品类标配
3. `[品类名] + 常见设计/标准做法` — 建立品类基线

**不需要搜索的情况**：原型清晰、做法概述足够判断。不要为了搜索而搜索。

---

### Step 4: 三层拆解

从 mechanism-analyse 报告中的机制数据出发，将发现分配到三层。

#### 原型层

从战斗/任务/养成/关卡四大系统的 ✓ 项中提取核心循环特征，与品类原型词典匹配的原型对比：
- 核心循环是否与原型一致？
- 乐趣来源是否与原型一致？
- 长线驱动力是否与原型一致？

结论：原型是 [原型名] / 原型有变化（说明哪里变了）

#### 创新层

**候选创新筛选**——按以下三步从报告中提取候选，不要自己"找异常"，而是让报告的数据引导你：

**第一步：读报告中的高信号区域**（这些区域已经帮你标出了重要机制）
- **综合叙述 → "该游戏的机制特色"**：报告作者总结的核心差异点，每一条都是候选
- **综合叙述 → "Top 5 值得吸收的机制"**：被标为 HIGH 的机制，尤其关注理由中提到"解决痛点"或"品类首创"的
- **综合叙述 → "品类对比观察"**：与竞品的差异点，直接指向可能的结构创新
- **机制清单表格 → 优先级为 HIGH 的子模块**：逐个审查其做法概述，判断是品类标配还是结构层创新

**第二步：补充扫描表格中的 ~ 标记项**
- 标记为 ~（部分存在/变体实现）的子模块，意味着做法偏离了通用描述，可能是创新候选
- 重点关注做法概述中出现"无传统""取消了""改为""独创"等措辞的子模块

**第三步：汇总候选清单**
- 将以上两步收集的所有候选机制列出（通常 5-10 个）
- 然后逐个过创新门槛和品类标配排除，最终筛选出真正的结构创新

**结构创新硬性判定门槛**（3 条全部满足才算结构创新）：

1. **玩家无需被告知就能自然感知到差异** — 不是藏在系统深处的数值调整
2. **影响的是体验节奏或结构，不只是数值或单点机制** — 不是"加了个新系统"就算
3. **去掉这个变化后，游戏体验会明显退化到更接近老原型的状态** — 它是区分度的来源

**每条门槛必须写一句论证理由**，不能只打 ✓。例如：
- 可感知：✓ "玩家每次死亡后立刻触发新对话，无需任何提示就能感受到'死了反而有新内容'"
- 结构性：✓ "将叙事进度绑定在死亡循环上，改变了玩家对'失败'的情感结构"
- 区分度：✓ "去掉这个设计，Hades 退化为标准无剧情肉鸽，与 Dead Cells 无本质区别"

不满足以上条件的变化，归入包装层的"包装设计"或"常规迭代"，不算结构创新。

**品类标配排除**：将候选创新与品类原型词典中该原型的"标志特征"对照。如果这个做法在原型的标志特征中已经明确列出，则不算结构创新，算品类标配。（详见第三节）

#### 包装层

包装层关注的是玩家"第一眼新感"——题材、美术、UI、叙事包装等表达层变化。

**数据来源**（报告中有多个地方包含包装层信息，不要遗漏）：

1. **基础信息**：游戏名、品类描述中的题材/世界观关键词
2. **综合叙述**：机制特色和品类对比中常提到美术/叙事/表达层差异
3. **机制清单中与表达层相关的子模块**（这些子模块的"做法概述"列包含丰富的包装层信息）：
   - **1.6 战斗反馈**：打击感、音效、视觉特效风格 — 属于表达层的"体感包装"
   - **9.1 HUD 设计**：界面风格、信息呈现方式 — 属于表达层的 UI 包装
   - **9.2 引导/教学系统**：叙事方式、教学包装手法 — 属于表达层的体验入口包装
   - **9.3 菜单/界面导航**：功能组织方式（如空间化 vs 菜单化） — 属于 UI 表达
   - **8.6 外观/皮肤系统**：角色/武器外观体系 — 属于视觉表达
   - **2.1/2.2 主线/支线任务**：叙事载体和呈现方式（过场/对话/环境叙事） — 属于叙事包装

**输出要点**：概括包装换在哪里（1-2 句话），点明是题材新、美术新、叙事方式新还是 UI 新。

---

### Step 5: 创新定性

基于三层拆解结果，给出一个主要判断（四选一）：

**换皮无创新**
品类原型没变，主要变化在表达层。
- 题材、美术、包装、卖点讲法变了
- 但核心循环、驱动力、品类原型都没明显变化

**原型+结构创新**
品类原型没变，但结构层出现了显著变化。
- 核心循环本质仍旧熟悉
- 但体验节奏、组织方式、门槛、爽点、传播性明显不同
- 玩家能真实感受到"老东西变好喝了"

**品类突破**
底层原型发生了明显变化。
- 核心循环变了
- 乐趣来源组合变了
- 长线驱动力变了
- 原有品类归属开始不够解释它

**硬性条件：核心循环、乐趣来源、长线驱动力三者中至少两项发生明显变化。**

**伪创新**
看起来有新意，但包装、结构创新、品类原型都没有形成真正有效变化。
- 营销说法新、单点机制新
- 但真实体验差异很弱
- 很难形成长期留存或市场优势

## 三、品类标配排除指南

### 核心问题
"这个做法在同品类游戏中有多常见？"

### 排除方法
将机制做法与品类原型词典中匹配原型的"标志特征"和"核心循环"逐项对照：
- 如果该做法在原型的标志特征中已列出 → **品类标配**，不是结构创新
- 如果该做法不在标志特征中，且符合创新 3 条门槛 → **可能是结构创新**

### 判断示例

| 游戏做法 | 匹配原型 | 判断 | 理由 |
|----------|---------|------|------|
| 卡牌RPG有抽卡系统 | 卡牌RPG-like | 品类标配 | 标志特征已包含"抽卡获取角色" |
| 卡牌RPG改为选择式获取 | 卡牌RPG-like | 可能是结构创新 | 改变了角色获取的结构和体验 |
| SLG有联盟系统 | COK-like | 品类标配 | 标志特征已包含"联盟协作" |
| SLG联盟允许双重间谍 | COK-like | 可能是结构创新 | 创造了新的博弈维度 |
| BR有缩圈机制 | BR-like | 品类标配 | 标志特征已包含"缩圈" |
| BR取消缩圈改用事件驱动 | BR-like | 可能是结构创新 | 从根本上改变了节奏结构 |

### 关键原则
结构创新不是"有什么"，而是"用不同的方式做了什么"。**存在性不够，差异性才算。**

## 四、评价卡片输出模板

```markdown
# [游戏名] 创新评价卡片

**品类原型**：[一句话，引用品类原型词典中的原型名称]
**包装**：[一句话，说清包装层换了什么]
**结构创新**：[逐条列出通过 3 条门槛的结构层变化；没有则写"无明显结构创新"]
**判断**：换皮无创新 / 原型+结构创新 / 品类突破 / 伪创新
**置信度**：高 / 中 / 低

## 机制证据

**原型判据**（支撑原型识别的关键子模块）：
- [子模块名]（[系统]）：[做法概述摘要] → 与 [原型名] 的 [标志特征] 一致/不一致
- ...

**创新判据**（通过 3 条门槛的创新机制）：

| 创新机制 | 来源子模块 | 做法概述 | 可感知（论证） | 结构性（论证） | 区分度（论证） |
|----------|-----------|----------|---------------|---------------|---------------|
| [名称] | [子模块] | [概述] | ✓ [一句话理由] | ✓ [一句话理由] | ✓ [一句话理由] |

**品类标配排除**（看似特别但实为品类标配的机制）：
- [子模块名]：[做法概述] → 排除原因：[原型名]标志特征已包含

> 一句话结论：本质上这是一个 ______ 品类原型的产品，通过 ______ 包装，[有/没有]在结构层做出有效结构创新。
```

## 五、语气与表达

### 开场
先给一句核心判断，再展开证据。不要一上来铺背景。

### 语气
直接、锋利、可用于讨论会。

**优先用**：
- 本质上、第一眼、拆开看、长期看
- 真正不同的地方、不在于……而在于……
- 这个机制解决的是……问题

**不要用**：
- 赋能、抓手、方法论闭环、生态协同、范式迁移
- 空洞的形容词（"很好的"、"不错的"）

### 结构
- 评价卡片用 Markdown 输出
- 关键判断用 **加粗** 标出
- 创新判据用表格呈现
- 一句话结论用引用块

## 六、禁止行为

- **禁止**自行扫描游戏机制（这是 mechanism-analyse 的工作）
- **禁止**捏造没有信息来源的机制细节
- **禁止**修改创新 4 类型分类或发明新类型/子类型
- **禁止**添加商业意义分析（点击率/转化率/ARPU/留存等）
- **禁止**添加风险分析章节
- **禁止**使用数值评分系统（1-10 打分等）
- **禁止**将品类标配标记为结构创新
- **禁止**输出 HTML 格式（纯 Markdown）
- **禁止**在没有 mechanism-analyse 报告的情况下凭空判断
