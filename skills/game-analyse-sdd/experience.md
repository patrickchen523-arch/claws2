# game-analyse-sdd 经验总结

## 技能简介
使用此技能对游戏进行标准化拆解分析。适用于用户要求分析游戏玩法、拆解游戏系统、评估游戏体验等场景。

按照标准拆解文档框架输出，包含：游戏基础信息、X-Statement、核心支柱、核心体验等模块。

## 使用经验

### 1. 标准化拆解框架

**X-Statement 公式**：
```
[题材][核心玩法]融合[次级玩法]与[养成要素]，玩家操控[角色类型]，在[场景类型]中使用[道具系统]构建独特[策略体系]。
```

**核心支柱分析**：
- 支柱 N：[具体体验]
  - 体验：描述玩家感受到的价值
  - 设计手段：实现该体验的方法
  - 具体参数：数字化/规则化/集合化

### 2. 搜索策略

**必搜关键词**：
- `[游戏名] + 评测/评测视频`
- `[游戏名] + 游戏机制/detailed breakdown`
- `[游戏名] + 社交/匹配/玩家互动`（社交维度必问）
- `[游戏名] + 商业化/营收模式`

**数据来源优先级**：
1. 专业游戏媒体：IGN、GameSpot、Polygon（设计分析更有深度）
2. Reddit 玩家讨论
3. 知乎/评测文章
4. Fandom Wiki（更关注数值攻略，缺乏设计层面洞察）

### 3. 社交维度必问问题

拆解任何游戏时，都应该问：
1. **玩家如何相遇/匹配？** - 随机匹配？好友列表？大厅？
2. **核心社交场景是什么？** - 交易？偷菜？展示？协作？
3. **如何保证活人感？** - 匹配机制？在线状态？随机刷新？

### 4. 品类理解框架

| 品类 | 重点关注 |
|------|----------|
| Roguelike | BD构筑、死亡转化、多人协作 |
| 策略桌游 | 资源权衡、行动规划、多轨道 |
| ARPG | 战斗反馈、成长曲线、技能联动 |
| 大逃杀/竞技 | 社交互动、队伍配合 |
| 模拟经营 | 约束设计、反馈循环、失败恢复 |

### 5. 工作流

1. 用户说"拆解 XXX"
2. 使用 web_search + web_fetch 收集信息
3. 按照 SDD 框架输出分析
4. 等待用户确认入库机制
5. 使用 game-entry-formatter 生成入库

## 常见问题

### Q: 搜索结果不够深入
**A**: 增加专业媒体搜索（IGN、Polygon），结合 Reddit 玩家讨论

### Q: 遗漏社交维度
**A**: 强制添加"社交维度必问问题"检查

### Q: X-Statement 不够准确
**A**: 先写核心玩法，再逐步添加题材、养成等修饰词
