# Game-Analyse-SDD 游戏分析技能

## 核心规则
- 结构化对齐： 必须严格遵循提供的"标准化拆解文档"的框架进行输出
- 聚焦使游戏成立的核心支柱（离开该系统游戏体验大打折扣）
- 观点来源：可以借鉴网络上已有对该游戏的分析，但需要注明来源网址
- 系统性分析： 分析任何一个单一系统时，必须结合相关的系统一起看

## 执行流程

### 第一步：收集游戏基础信息
- 搜索整理：开发者、开发者背景、上线时间、销量/收入
- 格式："《XX》是由XX公司自主研发的一款XX游戏，公司背景是...，代表作有...，于XX年XX月发布..."

### 第二步：分析X-Statement公式、玩法原型
- 搜索：拆解测评文章、官网介绍、用户讨论
- 输出格式：【基础信息】+【品类】+【核心差异点/独特卖点】
- 玩法原型：一句话描述核心体验

### 第三步：拆解游戏支柱（3-4个）
格式：支柱X：【支柱体验】-> 【设计手段】-> 【具体参数】

参数类型：
- 数字化：具体数值、百分比
- 规则化：触发条件 + 响应行为
- 集合化：列举实现手段/元素

### 第四步：拆解核心体验
- 以秒为单位：【镜头视角】+【玩家行为】+【乐趣】
- 以分钟/小时为单位：游戏节奏和用户体验流程

## 禁止行为
- 禁止重点放在边缘系统
- 禁止捏造数据（必须注明来源或说明"缺乏数据支持"）
- 禁止幻觉和瞎编
