# 游戏机制条目生成器

将游戏拆解分析结果格式化为符合 jizhi 机制库标准的 Markdown 条目。

## 适用场景

- game-analyse-sdd 拆解完成后，需要生成入库格式
- 用户确认哪些支柱可以入库后，生成最终条目

## 输出格式模板

```markdown
### [游戏名] ([英文名]) - [日期]

**X-Statement**：[一句话描述，包含题材、核心玩法、次级玩法、养成要素、角色类型、场景类型、道具系统、策略体系]

**核心支柱**：
1. **[支柱名称]** - [一句话描述]
   - 体验：[玩家感受到的价值]
   - 设计手段：[实现该体验的方法]
   - 具体参数：[数字化/规则化/集合化]

2. **[支柱名称]** - [一句话描述]
   ...

**玩法原型参考**：
- [品类继承]：[参考游戏1] / [参考游戏2]
- [子系统继承]：[参考游戏]

**基础信息**：
- 开发者：[开发者名称]
- 品类：[品类组合，如 FPS + Roguelite + RPG]
- 主题：[主题风格]
- 平台：[平台列表]
- 量化数据：[关键数据，如销量、在线人数等]
```

## 使用流程

1. 接收 game-analyse-sdd 的分析结果
2. 根据用户确认的入库项，筛选需要的内容
3. 按上述模板格式化输出
4. 用户确认后，提示直接复制到 jizhi 的 MEMORY.md

## 注意事项

- 保持 X-Statement 在一行内，长度约 50-80 字
- 核心支柱数量控制在 3-5 个
- 具体参数必须包含数字化指标（数量、比例等）
- 玩法原型要明确说明"旧酒"（传承了什么）和"新瓶"（创新了什么）
