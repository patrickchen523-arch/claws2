# 游戏机制条目生成器

将游戏拆解分析结果格式化为符合 jizhi 机制库标准的 Markdown 条目。

## 适用场景

- game-analyse-sdd 拆解完成后，需要生成入库格式
- 用户确认哪些支柱可以入库后，生成最终条目

## 入库格式模板

### 机制入库模板
| 字段 | 说明 | 示例 |
| ----- | ----------------------- | ---------------------- |
| 机制名称 | 标准化命名：[品类]-[游戏名]-[具体模块] | FPS-无畏契约-技能系统 |
| 核心标签 | 三维标签：形式+体验+品类 | 技能组合/局内成长/FPS |
| 一句话解释 | 用生活/商业类比讲清楚 | "类似MOBA的技能+FPS枪法=策略深度" |
| 机制描述 | 详细规则和交互流程（核心设计点要详细描述） | ... |
| 触发条件 | 机制何时生效 | ... |
| 解决痛点 | 解决什么负面体验 | ... |
| 体验目标 | 提供什么正面情绪 | ... |
| 相似机制 | 跨游戏同类参考 | ... |
| 演示素材 | GIF或视频链接（可选） | ... |

### 游戏入库模板
| 字段 | 说明 | 示例 |
| ----- | ----------- | ----------------- |
| 游戏名称 | 官方名称（中文/英文） | 无畏契约 / Valorant |
| 一句话对标 | 用已知游戏类比核心特色 | "CS的枪法+守望先锋的英雄技能" |
| 游戏类型 | 基础分类 | FPS / MOBA / 卡牌 |
| 核心循环 | 简述单局怎么玩 | 搜点→对枪→安包/拆包→下一回合 |
| 操作门槛 | 评估上手难度 | 枪法要求高 / 策略为主 |
| 收录机制 | 关联该游戏的已入库机制 | 自动关联显示 |

## 完整分析报告模板（game-analyse-sdd 产出）

```markdown
### [游戏名] ([英文名]) - [日期]

**X-Statement**：[一句话描述，包含题材、核心玩法、次级玩法、养成要素、角色类型、场景类型、道具系统、策略体系]

**核心支柱**：
1. **[支柱名称]** - [一句话描述]
   - 体验：[玩家感受到的价值]
   - 设计手段：[实现该体验的方法]
   - 具体参数：[数字化/规则化/集合化]

2. **[支柱名称]** - [一句话描述]
   ...

**玩法原型参考**：
- [品类继承]：[参考游戏1] / [参考游戏2]
- [子系统继承]：[参考游戏]

**基础信息**：
- 开发者：[开发者名称]
- 品类：[品类组合，如 FPS + Roguelite + RPG]
- 主题：[主题风格]
- 平台：[平台列表]
- 量化数据：[关键数据，如销量、在线人数等]
```

## 使用流程

1. 接收 game-analyse-sdd 的分析结果
2. 根据用户确认的入库项，筛选需要的内容
3. 按上述机制入库模板格式化输出（每个机制单独一个条目）
4. 用户确认后，通过 @Jysmechanism_bot 发送到 jizhi 群

## 注意事项

- 机制描述中的核心设计点要详细描述（如降门槛的具体实现方式）
- 核心标签采用三维标签：形式+体验+品类
- 保持一句话解释在一行内，长度约 20-40 字
- 核心支柱数量控制在 3-5 个
- 具体参数必须包含数字化指标（数量、比例等）
- 玩法原型要明确说明"旧酒"（传承了什么）和"新瓶"（创新了什么）
