# 游戏机制条目生成器

将游戏拆解分析结果格式化为符合 jizhi 机制库标准的 Markdown 条目。

## 适用场景

- game-analyse-sdd 拆解完成后，需要生成入库格式
- 用户确认哪些支柱可以入库后，生成最终条目

---

## 机制库：设计详情标准卡 (Mechanism Card)

*核心资产库，采用高颗粒度拆解，确保不仅"能看懂"，还能"落地参考"。*

### 模板

```markdown
# [品类]-[游戏名]-[具体模块]

## 基础信息

| 字段 | 内容 |
| :--- | :--- |
| 案例名称 | [品类]-[游戏名]-[具体模块] |
| 来源游戏 | 🔗 *游戏名* |
| 核心标签 | 形式+体验+品类 |

---

## 通俗转译

**一句话解释**：
> 用生活/商业类比，供非策划人员快速理解

---

## 机制拆解

**机制描述**：
> 详细的规则说明，包含交互流程

**触发条件**：
> 机制生效的时机或前置条件

---

## 设计意图

**解决痛点**：
> 该机制是为了解决什么负面体验？

**体验目标**：
> 该机制希望提供什么正面情绪？

---

## 关联参考

**相似机制**：
> 跨游戏的同类案例，方便横向对比

---

## 多媒体

**演示素材**：
> GIF动图或视频链接（必填）
```

### 字段规范

| 字段模块 | 字段名称 | 说明与填写规范 |
| :--- | :--- | :--- |
| 基础信息 | 案例名称 | 标准化命名：[品类]-[游戏名]-[具体模块] |
| | 来源游戏 | [关联字段] 指向游戏库，点击跳转 🔗 *游戏名* |

| 机制拆解 | 机制描述 | 详细的规则说明，包含交互流程 |
| | 触发条件 | 机制生效的时机或前置条件 |

## 游戏库：背景扫盲卡 (Context Card)

*辅助库，重点在于"扫盲"和"对标"，让不了解该游戏的人也能看懂机制的生效环境。*

### 模板

```markdown
# 游戏名 / Game Name

## 基本信息

| 字段 | 内容 |
| :--- | :--- |
| 游戏名称 | 游戏官方名称（中文/英文） |
| 一句话对标 | 核心字段。用已知游戏进行类比（如：带枪的DOTA2） |
| 游戏类型 | 基础分类（如：TPS / MOBA / 卡牌） |

---

## 玩法环境

**核心循环**：
> 简述单局怎么玩（如：搜集->跑圈->对枪）

**操作门槛**：
> 评估受众群体的上手难度

---

## 关联资产

**收录机制**：
> [自动关联] 显示机制库中所有属于该游戏的条目
```

### 字段规范

| 字段模块 | 字段名称 | 说明与填写规范 |
| :--- | :--- | :--- |
| 基本信息 | 游戏名称 | 游戏官方名称（中文/英文） |
| | 一句话对标 | 核心字段。用已知游戏进行类比 |
| | 游戏类型 | 基础分类（如：TPS / MOBA / 卡牌） |
| 玩法环境 | 核心循环 | 简述单局怎么玩 |
| | 操作门槛 | 评估受众群体的上手难度 |
| 关联资产 | 收录机制 | 自动关联显示机制库中所有属于该游戏的条目 |

---

## 使用流程

1. 接收 game-analyse-sdd 的分析结果
2. 根据用户确认的入库项，筛选需要的内容
3. 按上述模板格式化输出
4. 用户确认后写入 jizhi workspace：
   - 机制：`/root/.openclaw/agents/jizhi/workspace/mechanisms/[游戏英文名]/[机制名].md`
   - 游戏：`/root/.openclaw/agents/jizhi/workspace/games/[游戏英文名].md`

## 注意事项

- **一句话解释**：必填，用生活/商业类比讲清楚
- **演示素材**：机制卡必填，提供GIF或视频链接
- **核心标签**：形式+体验，如 "Roguelike三选一 局内构建 Moba"
- **来源游戏**：格式为 🔗 *游戏名*，与游戏库关联
- **收录机制**：格式为 [品类]-游戏名-机制名
