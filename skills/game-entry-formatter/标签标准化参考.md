# 机制库标签标准化参考文档

> 本文档用于指导机制卡的标签标准化工作
> 标签分为三个层级：玩法层级、机制设计层级、痛点层级

---

## 一、玩法层级标签（必填）

### 1. 对抗类型

| 标签 | 说明 | 示例 |
|-------|------|------|
| PVE | 玩家 vs 环境/AI | 副本、Boss战 |
| PVP | 玩家 vs 玩家 | 天梯、排位 |
| PvPvE | 先PVE后PVP | 搜打撤、逃离塔科夫 |
| GVG | 公会 vs 公会 | 公会战、阵营战 |
| 异步竞技 | 非实时对抗 | 异步PVP、离线收益比较 |

### 2. 玩法人数规模

| 标签 | 说明 | 示例 |
|-------|------|------|
| 单人 | 1人游戏 | 单机RPG |
| 双人 | 2人 | 本地双人、合作 |
| 小组多人 | 3-4人 | 三人组队、四人副本 |
| 中型多人 | 5-10人 | 5v5、10人副本 |
| 大型多人 | 10+人 | 公会战、MMO战场 |

**格式**：单局X人 / 单队X人

### 3. 内容消耗模式

| 标签 | 说明 | 示例 |
|-------|------|------|
| 一次性 | 线性流程，关卡制 | 单机剧情 |
| 周期性 | 赛季、轮换活动 | 赛季通行证、月度活动 |
| 可重复 | 无尽刷宝 | Roguelike、刷宝 |
| UGC | 用户生成内容 | 地图工坊、编辑器 |

### 4. 生命周期阶段

| 标签 | 说明 | 示例 |
|-------|------|------|
| 新手引导期 | 教程、前期成长 | 新手任务、引导关卡 |
| 成长中期 | 中期BD构建 | 天赋树、装备积累 |
| Endgame | 毕业、高难度 | 高难副本、排名竞争 |

### 5. 操作属性

| 标签 | 说明 | APM参考 |
|------|------|----------|
| 即时操作 | 实时操作 | APM>60 |
| 半即时操作 | 有操作窗口 | APM 15-60 |
| 回合操作 | 回合制思考 | APM<15 |
| 放置操作 | 挂机、单手操作 | APM<5 |

### 6. 时间属性

| 标签 | 说明 | 示例 |
|-------|------|------|
| 长时间 | >60分钟/局 | 硬核生存、深度策略 |
| 中规中矩 | 10-60分钟/局 | 标准MOBA、RPG副本 |
| 碎片化 | 3-10分钟/局 | 休闲手游、短对局 |
| 高度碎片化 | 1-3分钟/局 | 地铁对局、等待间隙 |

### 7. 玩法重度

| 标签 | 说明 | 示例 |
|-------|------|------|
| 核心玩法 | 主体玩法 | 战斗系统、策略深度 |
| 副玩法 | 辅助玩法 | 小游戏、抽卡系统 |
| 轻度玩法 | 休闲内容 | 叙事、对话、社交 |

### 8. 玩法独立性

| 标签 | 说明 | 示例 |
|-------|------|------|
| 高耦合 | 核心数值系统 | 经济、装备、BD |
| 低耦合 | 独立小游戏 | 钓鱼、换装、聊天 |

### ⚠️ 重要规则

**遇到未收录的品类、情绪、交互关系时**：
- 暂时使用"其他"占位
- 发送消息给俊亦确认是否加入模板
- 确认后更新模板，再修改机制卡

### 情绪反馈判定标准

**重要规则**：请阅读机制描述，模拟你是玩家。选择强度最高的1个核心情绪。

**注意事项**：
- **关联性校验**：
  - 如果选了"烧脑"，请检查16乐趣里是否有"策略"或"解谜"
  - 如果选了"治愈"，请检查16乐趣里是否没有"竞技"和"挑战"
- **互斥检查**："爽快感"与"烧脑"通常不同时作为核心反馈出现，请甄别主次

| 标签 | 判定题 | 心理/生理表现 |
|------|--------|---------------|
| **爽快感** | 玩家是否在短时间内进行了高频的操作输入，且伴随大量的视觉特效反馈？ | 肾上腺素飙升，感到"宣泄"和"停不下来" |
| **惊喜随机** | 结果是否超出了玩家的预期（由于系统概率），且该结果显著利好于玩家？ | 尖叫、截图分享、"今天运气真好" |
| **紧张感** | 玩家是否处于高风险环境下（如资源枯竭、血量见底、限时倒计时），且一步操作失误就会导致巨大损失？ | 屏住呼吸、手心出汗、心跳加速 |
| **烧脑** | 玩家是否陷入了长时间的沉默思考、反复对比不同方案，甚至需要动用纸笔或计算器？ | 眉头紧锁，在"我懂了"的那一刻爆发 |
| **恐怖/微恐** | 机制是否利用了未知感、幽闭环境、突发音效或视觉上的扭曲来让玩家感到"不安全"？ | 肌肉紧绷、下意识地想缩在屏幕远端 |
| **节奏沉浸** | 玩家是否陷入了一种重复但有节奏的行为（如音游、钓鱼、循环刷怪），导致忘记了时间的流逝？ | 感到"心流"，周围的环境噪音被自动过滤 |
| **治愈/放松** | 机制是否排除了"失败"和"死亡"的可能，且美术/音乐表现柔和，不给玩家任何压迫感？ | 肩膀放松、深呼吸、感到心情平静 |
| **成就感** | 玩家是否在完成了一个高难度目标（或长期积累任务）后，获得了一个象征荣誉的反馈？ | 如释重负、感到"所有的付出都值了" |

### 交互关系判定标准

**核心思路**：聚焦于"玩家行为对另一个人的影响结果"。

**甄别准则**：
- 如果同时存在合作与对抗（如：组队PVP），标签应标为"合作"、"对抗"
- 如果只是在主城能看见人，但机制是自己打怪，请务必甄别是否为"无交互"
- "欺骗"是高阶标签，除非机制卡里明确提到了"身份"、"伪装"、"说谎"，否则不要轻易打这个标签

| 标签 | 判定题 | 核心逻辑 |
|------|--------|----------|
| **合作** | 如果其中一名玩家表现极差，另一名玩家是否会直接导致任务失败或利益受损？ | 利益共同体。你的成功就是我的成功。 |
| **对抗** | 玩家之间是否可以互相造成伤害、位移、资源扣除，或者存在"你多得一个，我就少得一个"的零和博弈？ | 利益对立面。你的损失就是我的收益。 |
| **欺骗** | 机制是否涉及"隐藏身份"、"虚假信息传递"或"背叛机制"？ | 信息差博弈。如果玩家之间完全诚实，这个机制是否会立刻失效？ |
| **交易/互惠** | 玩家是否通过"交出某样东西"来换取"另一样东西"，且这个过程是自愿且双赢的？ | 等价交换。双方在互动后，资源总量不变或优化，但分配方式改变。 |
| **无交互** | 在该机制进行时，其他玩家的存在（无论是操作还是数值）是否对当前玩家的体验产生0影响？ | 平行时空。即使服务器里只有我一个人，这个机制的玩点也完全不变。 |

### 对抗类型判定标准

**甄别补充原则**：
- **第一优先级（看目标）**：只要能打到真人，先标记为PVP或PvPvE
- **第二优先级（看同步性）**：如果能打到真人，但对方是自动出招/离线，强制修正为异步竞技
- **第三优先级（看规模）**：如果必须以组织名义报名，且有公会积分，标记为GVG
- **特殊修正**：如果玩家只是在比谁打AI打得快，但玩家之间不能互殴，标记为[PVE]+[间接竞争]，而不是PVP

| 标签 | 判定题 | 甄别点 |
|------|--------|--------|
| **PVE** | 胜负手是否完全掌握在玩家对AI或系统规则的处理上？ | 即使有其他玩家在场，只要目标是"打怪/闯关"且没有互相伤害，就属于PVE |
| **PVP** | 玩家是否必须通过直接消耗/击败另一个真实玩家的操作或数值来获得胜利？ | 必须是实时、直接的对抗 |
| **PvPvE** | 玩家在执行PVE任务（如打怪、搜物资）的同时，是否随时面临被其他玩家攻击并夺取收益的风险？ | 环境（AI）和玩家同时构成威胁，且两者利益纠缠 |
| **GVG** | 机制的胜负结果是否直接计入"公会/俱乐部/帮派"的集体荣誉或资源池，且参与者必须具备组织身份？ | 强调"组织"对战 |
| **异步竞技** | 当你攻击或挑战对手时，对手是否必须在线并由真人操作？ | 如果你打的是"离线镜像"、"录像"或"排行榜上的数据"，则属于异步竞技 |
| **阵营对抗** | 玩家是否被划分为两个或多个超大型长期固定组织，且对抗规模覆盖全服？ | 对抗通常不限人数，是全服性质的利益争夺 |
| **非对称** | 是否符合以下任一特征：人数不相等；双方操作维度/技能体系/视角完全不同；获胜条件完全不同？双方是否在玩"两个不同的游戏"？ | 典型例子：1v4逃生vs监管、imba vs 普通人 |

**重要规则**：如果发现超过2个乐趣符合要求，请判断最重要的两个。

**判定流程**：
1. 先回答16道判定题
2. 筛选出"是"的选项
3. 排序：选出1个最主要的（支撑机制核心目标的）和1-2个辅助的

**示例**：
- 机制："需要玩家组队在1秒内同时按下按钮并击败BOSS"
- 回答判定题：操作(是)、社交(是)、挑战(是)
- 结论：核心乐趣-挑战，辅助-社交、操作

| 标签 | 判定题 | 甄别点 |
|------|--------|--------|
| **解谜/破解** | 玩家是否需要通过观察环境、推导逻辑规律来获得一个"唯一解"或"特定过关方式"？ | 重点在于信息不对称的消除。一旦知道了答案，第二次玩是否就失去了90%的乐趣？ |
| **策略** | 玩家是否需要在多个"正确方案"中，根据资源、环境或对手行为，通过长线规划选出"最优解"？ | 重点在于动态博弈和资源分配。它通常不只有唯一解，且需要持续的决策输入。 |
| **考究** | 该机制是否提供了大量的文字、物品描述或隐藏细节，且这些内容不直接影响胜负，但会补完世界观？ | 玩家是否在玩"拼图"，通过碎片信息去还原真相。 |
| **操作** | 玩家的"反应速度"、"肌肉记忆"或"输入精准度"是否是决定胜负的核心因素？ | 哪怕玩家脑子里知道怎么赢，但如果手跟不上，是否就会失败？ |
| **挑战** | 机制是否刻意设置了极高的失败门槛，要求玩家必须经历"失败-学习-再尝试"的反复循环？ | 重点在于"克服逆境"。它是针对玩家意志力的磨炼，而非简单的杀时间。 |
| **竞技** | 玩家的成就感是否直接来源于"在规则下击败了另一个真实人类"或"在排行榜上压倒了他人"？ | 强调横向对比。如果没有其他玩家作为参照物，这个乐趣是否消失？ |
| **代入/剧情** | 玩家是否被要求扮演一个特定角色，且该角色的情感、对白或命运发展是吸引玩家玩下去的主因？ | 机制是否在服务于"讲故事"。如果把文字和美术全删掉只留数值，玩家是否还会觉得有趣？ |
| **创造** | 玩家是否利用游戏提供的原件，输出了一个"世界上独一无二"的成果（如关卡、建筑、逻辑回路）？ | 玩家的身份是否从"消费者"变成了"生产者"。 |
| **模拟现实** | 机制是否严格遵循现实世界的物理规律、职业逻辑或社会规则，以追求"真"而非"爽"？ | 如果现实中能做这件事，游戏中是否也采用了类似的步骤和限制？ |
| **养成/装扮** | 玩家是否对某个对象（角色、基地、宠物）投入了大量"情感或审美支出"，目的是让它看起来更好看或更符合个性？ | 重点在于"情感寄托"和"外在表现"。 |
| **成长变强** | 玩家是否通过消耗资源，获得了角色数值（攻击力、血量等）的永久提升？ | 重点在于"效率的提升"。以前打不过的怪，现在能一刀秒，这就是变强。 |
| **收集** | 机制是否提供了一个"清单"或"图鉴"，并鼓励玩家将其中缺失的空格填满？ | 重点在于"占有欲"。玩家是否为了凑齐一套不一定有实际用途的东西而努力。 |
| **盈利/赚钱** | 机制的核心目标是否是"低买高卖"、"资产增值"或"建立经济帝国"？ | 数字的增长本身就是目的，而不仅仅是变强的手段。 |
| **探索/高自由** | 玩家是否在没有明确指引的情况下，因为好奇心驱使去到了地图的死角，并获得了意外的反馈？ | 重点在于"未知"。玩家是否在走策划没有在小地图上标出来的路。 |
| **随机** | 结果是否由系统的随机数（RNG）决定，导致玩家在操作完全一致的情况下获得截然不同的结果？ | 重点在于"赌徒心理"。是否有"出货了"或"太非了"的情绪波动。 |
| **社交** | 玩家是否必须（或非常倾向于）通过与其他玩家交谈、组队或交换资源来达成目标？ | 重点在于"人与人的连接"。如果把所有玩家换成AI机器人，乐趣是否会大幅下降？ |

## 三、标签存储格式

### 机制卡存储格式

在机制卡中使用以下格式存储标签：

```markdown
## 标签信息

### 玩法层级
```

### 内容消耗模式判定标准

**核心思路**：聚焦于"新鲜感消失的速度"以及"重玩的驱动力"。

**易错点纠正**：
- **刷副本 ≠ 可重复**：如果玩家重复游玩只是为了刷数值（等级/装备），且过程枯燥无味，归类为"周期性"或"持续运营"。如果每一局过程都有博弈和变化，归类为"可重复"
- **整体≠部分**：一个游戏的持续运营，不代表它的每个机制都是持续运营。例如《原神》的地图探索是"一次性"的，但整体更新机制是"持续运营"

| 标签 | 判定题 | 逻辑 |
|------|--------|------|
| **一次性** | 玩家在通关或知晓结果后，如果再次游玩，由于机制或内容完全没有变化，其乐趣是否会下降80%以上？ | 消耗即毁灭。通常指剧情、特定解谜关卡、纯叙事流程 |
| **周期性** | 该机制是否有明确的重置时间（日、周、月、赛季）？且玩家重玩的动力是否主要来源于"刷新后的奖励"？ | 任务制。为了拿工资（资源），玩家愿意定期重复 |
| **可重复** | 即使没有奖励，该玩法是否依然具有极高的策略深度、操作上限或随机组合，使得每一局的过程都截然不同？ | 规则驱动。每一局都是新体验。例如：MOBA单局、肉鸽随机层 |
| **持续运营** | 该机制是否需要官方高频更新美术素材、新角色、新地图或新剧本才能维持热度？ | 堆料驱动。官方更新多快，玩家就活多久 |
| **UGC** | 该玩法的内容主体（地图、关卡、规则）是否由玩家自主上传？ | 众包驱动 |

### 生命周期阶段判定标准

**核心思路**：聚焦于"准入门槛"和"它在玩家日程表里的位置"。

**补充逻辑**：
- **新手引导期**：玩家"跟着走"
- **成长中期**：玩家"为了强"（数值驱动）
- **Endgame**：玩家"为了爽/为了赢/为了秀"（荣誉/社交/自我实现驱动）

**修正逻辑**：
- "Endgame"的伪装：不要因为"很难"就标Endgame。高难副本如果掉落强化材料，实际是[成长中期]；如果只掉落荣誉头衔，才是[Endgame]
- 如果玩法贯穿全周期：寻找该机制的"质变点"

| 标签 | 判定题 | 逻辑 |
|------|--------|------|
| **新手引导期** | 该玩法是否伴随着大量的强制点击指引、剧情对白，且玩家几乎不可能在这里失败？最终奖励是否主要是"解锁了某个新系统"或"获得了一套基础套装"？ | 它是"门票"。如果玩家已经玩了50小时，他还会回来玩这个吗？（若不会，则是引导期） |
| **成长中期** | 玩家是否每天/每周必须完成该玩法，以获取升级角色数值或装备所需的材料？玩家追求的是否是"更快的通关速度"或"更优的刷怪效率"，而不是"全新的体验"？ | 它是"主食"。这是玩家在游戏20%到80%进度时花费时间最多的地方 |
| **Endgame** | 该玩法是否要求玩家必须达到一定门槛才能进入？反馈是否主要是"全服排名"、"独特的皮肤外观"或"极其复杂的社交协作"？ | 它是"终极追求"。玩家已经"到头"了，追求的是荣誉/社交/自我实现 |

### 操作属性判定标准

**逻辑补充（易错案例）**：
- **多自动战斗的RPG**：画面虽然在实时打斗（即时表现），但玩家只需要点一下"倍速"或"领取奖励"。这种情况必须标为"放置操作"或"半即时"，不能因为画面动得快就标"即时操作"
- **可暂停下指令的游戏**：如果"暂停思考"是核心体验的一部分，建议标为"回合操作"或"半即时"，以体现其"重决策、轻操作"的本质

| 标签 | 特征 | 判定题 | 典型 |
|------|------|--------|------|
| **即时操作** | 玩家输入与角色反馈几乎同步（<100ms），操作连续，输入频率决定效率 | 机制是否要求玩家具备"手眼协同"能力（瞄准、精准格挡、卡帧闪避）？ | FPS射击、动作割草、MOBA走位 |
| **半即时操作** | 存在自动攻击或自动寻路，但玩家需要在关键时刻点选技能、调整站位或控制冷却 | 角色是否会在没有指令的情况下自动执行基础行为？玩家的主要操作是否集中在"释放冷却技能"或"在特定节点改变角色状态"？ | 塔防放置、ARPG半自动模式、技能排队 |
| **回合操作** | 时间轴离散，系统等待玩家确认指令后才执行，决策时间通常不受限制 | 系统是否会无限期等待玩家做出决定（或者有非常长的单回合思考时间）？ | 战棋、策略卡牌、JRPG战斗 |
| **放置操作** | 战前策略布置，战斗过程玩家无法有效干预，支持后台挂机或离线产出 | 机制的核心乐趣是否在于"战前的策略布置"，而战斗过程中玩家无法进行任何有效干预？该玩法是否支持"后台挂机"或"离线产出"？ | 挂机游戏、自走棋战斗阶段、家园资源产出 |

### 时间属性判定标准

**辅助判定规则**：
- **强制时限判定**：该机制是否有硬性的单局倒计时？如果有，直接取该时长作为基准
- **"退出"代价判定**：如果玩家在进行该机制时突然断线或停止操作5分钟，他的进度损失会有多大？
  - 损失极小（随时存取）→ 极度碎片化/碎片化
  - 损失极大（无法重连/关卡重头）→ 中规中矩以上

**易错案例纠正**：
- **SLG建筑升级**：升级一座主城需要24小时，但玩家的交互时间仅有点击的一秒钟，应标为"极度碎片化"，不是"长时间"

| 标签 | 时长 | 典型 |
|------|------|------|
| **极度碎片化** | 1-3分钟 | 抽卡、领取奖励、SLG点建筑、简单对话 |
| **碎片化** | 3-10分钟 | 皇室战争单局、肉鸽一层、日常战斗任务 |
| **中规中矩** | 10-60分钟 | MOBA单局、吃鸡单局、大型副本分区 |
| **长时间** | >60分钟 | 大型团本、长流程策略对局、MMO阵营战 |

### 玩法重度判定标准

**定义**：
- **核心玩法**：游戏存在的理由。玩家每天必须投入60%以上精力和时间的机制
- **副玩法**：调节剂。用于缓解核心玩法的疲劳，或者作为赛季/限时活动出现
- **轻度玩法**：极低门槛。如签到、简单的对话、纯叙事表现

**判定题**：
- **产出唯一性**：如果玩家完全不玩这个机制，他是否还能正常提升角色的核心战力或推进主线？
  - 不能 → 核心玩法
  - 能，但变慢了 → 副玩法
- **心智占用**：游玩该机制时，玩家是否可以同时听播客、聊天或进行其他多线程任务？
  - 完全不能（必须全神贯注）→ 核心玩法/重度
  - 可以（甚至支持自动）→ 轻度玩法

### 玩法独立性判定标准

**定义**（为策划拆解服务，反映机制移植到其他项目的难度）：
- **高耦合**：机制与游戏的数值系统（攻击力、防御力、核心货币）深度绑定
- **低耦合**：机制逻辑相对独立。换一套美术资源和数值环境，逻辑依然跑得通（如内置消消乐、钓鱼、扫雷）

**判定题**：
- **经济流向**：该机制产生的奖励是否是游戏内的"通用货币"或"通用强化材料"？
  - 是（如产出金币、经验）→ 高耦合
  - 否（仅产生该玩法内的积分或皮肤奖励）→ 低耦合
- **逻辑依赖**：如果把这个机制从代码里"切除"，游戏的其他功能是否会报错或失效？
  - 会失效 → 高耦合
  - 不会（像独立的小程序）→ 低耦合
- **数值共用**：玩法中是否直接套用了玩家在局外的等级、装备和属性加成？
  - 是 → 高耦合
  - 否（局内独立数值，如自走棋、肉鸽局内成长）→ 低耦合

**纠错逻辑**：避免"大厂迷信"
- 区分"入口"与"内核"：某个MMO里的赛马比赛不是核心玩法（不影响砍怪属性，逻辑独立）→ 副玩法 + 低耦合
