# 机制库标签标准化参考文档

> 本文档用于指导机制卡的标签标准化工作
> 标签分为三个层级：玩法层级、机制设计层级、痛点层级

---

## 一、玩法层级标签（必填）

### 1. 对抗类型

| 标签 | 说明 | 示例 |
|-------|------|------|
| PVE | 玩家 vs 环境/AI | 副本、Boss战 |
| PVP | 玩家 vs 玩家 | 天梯、排位 |
| PvPvE | 先PVE后PVP | 搜打撤、逃离塔科夫 |
| GVG | 公会 vs 公会 | 公会战、阵营战 |
| 异步竞技 | 非实时对抗 | 异步PVP、离线收益比较 |

### 2. 玩法人数规模

| 标签 | 说明 | 示例 |
|-------|------|------|
| 单人 | 1人游戏 | 单机RPG |
| 双人 | 2人 | 本地双人、合作 |
| 小组多人 | 3-4人 | 三人组队、四人副本 |
| 中型多人 | 5-10人 | 5v5、10人副本 |
| 大型多人 | 10+人 | 公会战、MMO战场 |

**格式**：单局X人 / 单队X人

### 3. 内容消耗模式

| 标签 | 说明 | 示例 |
|-------|------|------|
| 一次性 | 线性流程，关卡制 | 单机剧情 |
| 周期性 | 赛季、轮换活动 | 赛季通行证、月度活动 |
| 可重复 | 无尽刷宝 | Roguelike、刷宝 |
| UGC | 用户生成内容 | 地图工坊、编辑器 |

### 4. 生命周期阶段

| 标签 | 说明 | 示例 |
|-------|------|------|
| 新手引导期 | 教程、前期成长 | 新手任务、引导关卡 |
| 成长中期 | 中期BD构建 | 天赋树、装备积累 |
| Endgame | 毕业、高难度 | 高难副本、排名竞争 |

### 5. 操作属性

| 标签 | 说明 | APM参考 |
|------|------|----------|
| 即时操作 | 实时操作 | APM>60 |
| 半即时操作 | 有操作窗口 | APM 15-60 |
| 回合操作 | 回合制思考 | APM<15 |
| 放置操作 | 挂机、单手操作 | APM<5 |

### 6. 时间属性

| 标签 | 说明 | 示例 |
|-------|------|------|
| 长时间 | >60分钟/局 | 硬核生存、深度策略 |
| 中规中矩 | 10-60分钟/局 | 标准MOBA、RPG副本 |
| 碎片化 | 3-10分钟/局 | 休闲手游、短对局 |
| 高度碎片化 | 1-3分钟/局 | 地铁对局、等待间隙 |

### 7. 玩法重度

| 标签 | 说明 | 示例 |
|-------|------|------|
| 核心玩法 | 主体玩法 | 战斗系统、策略深度 |
| 副玩法 | 辅助玩法 | 小游戏、抽卡系统 |
| 轻度玩法 | 休闲内容 | 叙事、对话、社交 |

### 8. 玩法独立性

| 标签 | 说明 | 示例 |
|-------|------|------|
| 高耦合 | 核心数值系统 | 经济、装备、BD |
| 低耦合 | 独立小游戏 | 钓鱼、换装、聊天 |

| 标签 | 说明 |
|------|------|
| 时间成本-日常负担 | 每日任务压力 |

| 认知负担-信息过载 | 系统复杂 |
| 认知负担-学习曲线 | 上手难 |
| 内容枯竭-后期无聊 | 内容耗尽 |
| 内容枯竭-重复可玩性 | 重复内容多 |
| 社交压力-组队需求 | 必须组队 |
| 社交压力-野队配合 | 匹配队友 |
| 操控体验-手感差 | 反馈不足 |
| 操控体验-操作门槛 | 键位复杂 |

| 沉浸感-代入感差 | 角色塑造 |

| 自由度-限制过多 | 选择少 |
| 情感共鸣-角色扁平 | 角色无特色 |
| 情感共鸣-故事老套 | 剧情俗套 |

### 6. 特色机制（1-3个）

**格式**：每个机制单独一行，使用"特色机制-具体机制名"格式

示例：
```markdown
### 特色机制
- 特色机制-失衡系统
- 特色机制-装备带入带出
```

## 三、标签存储格式

### 机制卡存储格式

在机制卡中使用以下格式存储标签：

```markdown
## 标签信息

### 玩法层级
- 对抗类型：PVP
- 玩法人数规模：单局5人/单队5人
- 内容消耗模式：可重复
- 生命周期阶段：Endgame
- 操作属性：即时操作
- 时间属性：中规中矩（10-60分钟）
- 玩法重度：核心玩法
- 玩法独立性：高耦合

### 机制设计
- 原型品类：射击-搜打撤
- 16乐趣标签：策略、竞技、挑战
- 情绪反馈：紧张感、爽快感
- 交互关系：对抗
- 特色机制：撤离机制、装备带入带出

### 玩家痛点
- 挫败感痛点
- 经济成本痛点
```

### HTML显示格式

解析器只显示具体标签，不显示表头。例如：
- 玩法层级显示：`PVP | 5v5 | 可重复 | Endgame`
- 机制设计显示：`射击-搜打撤 | 策略、竞技 | 紧张感`
- 痛点显示：`时间成本-日常负担`

## 四、判断规则

### 判断顺序

1. **先判断玩法层级** - 根据游戏核心循环判断
2. **再判断机制层级** - 根据机制所属品类判断
3. **最后判断痛点** - 根据机制解决什么问题判断

### 判断技巧

- 如果网上搜索无法确认标签 → 标注"无法判断"
- 如果机制跨越多个分类 → 选择最核心的1-2个
- 如果机制同时服务多个玩法 → 用"/"分隔
